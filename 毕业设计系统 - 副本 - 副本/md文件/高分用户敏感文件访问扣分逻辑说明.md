# 高分用户敏感文件访问扣分逻辑说明

## 📋 功能概述

系统根据**员工信任分**和**行为风险等级**进行交叉决策，实现分级响应策略。高分员工打开敏感文件时，采用轻度惩罚机制，避免过度干扰正常工作。

---

## 🎯 实际案例

### 场景描述
- **用户ID**: 7
- **初始信任分**: 88.09分（高信任）
- **触发行为**: 打开 `C:\Users\HP\Desktop\未公开源码_核心算法.py`
- **检测结果**: 窗口标题命中敏感词"源码"
- **风险等级**: 高危（`behavior_risk = 'high'`）

### 执行结果
```
[2026-05-21 17:22:52] 上报成功 | 状态: success | 操作: warn | 信任分: 86.09
```

**扣分数值**: **2分**（88.09 → 86.09）

---

## 🔍 扣分逻辑详解

### 1. 风险判定流程

```python
# 步骤1: 检测敏感词
if is_sens:  # 检测到敏感词"源码"
    behavior_risk = 'high'  # 标记为高危行为
    
# 步骤2: 匹配决策矩阵
if trust_score > 80 and behavior_risk == 'high':
    # 高信任员工 + 高危行为
    action_taken = "warn"           # 仅告警，不弹窗
    alert_level = "medium"          # 中等级别告警
    new_trust_score -= 2            # ✅ 扣2分
```

### 2. 前端阻断执行

即使只是 `warn` 级别，前端仍然执行以下阻断措施：

1. **锁定文件权限**（使用 `icacls` 命令）
2. **关闭编辑器标签页**（模拟 Alt+F4）
3. **显示阻断弹窗**（全屏遮罩）
4. **上报事件到后端**

## 📊 完整的分级扣分策略

### 决策矩阵规则

| 信任分区间 | 高危行为 | 中危行为 | 低危行为 |
|-----------|---------|---------|---------|
| **>80分** (高信任) | warn (-2分) | pass (0分) | pass (0分) |
| **60-80分** (中信任) | confirm (-5分) | pass (0分) | pass (0分) |
| **<60分** (低信任) | confirm (-15分) | confirm (-5分) | pass (0分) |

---

### 详细说明

#### 1️⃣ 高信任员工（>80分）

**设计理念**: 信任但验证，给予宽容度

| 行为类型 | 操作 | 扣分 | 前端响应 |
|---------|------|------|---------|
| 敏感文件访问 | `warn` | -2分 | 锁定文件+关闭标签+弹窗 |
| USB插入 | `pass` | 0分 | 静默记录 |
| 正常办公 | `pass` | 0分 | 无响应 |

**示例**:
- 88分员工打开敏感文件 → 86分（-2分）
- 95分员工插入U盘 → 95分（不变）

---

#### 2️⃣ 中信任员工（60-80分）

**设计理念**: 谨慎监管，平衡安全与效率

| 行为类型 | 操作 | 扣分 | 前端响应 |
|---------|------|------|---------|
| 敏感文件访问 | `confirm_required` | -5分 | 二次确认弹窗 |
| USB插入 | `pass` | 0分 | 静默记录 |
| 正常办公 | `pass` | 0分 | 无响应 |

**示例**:
- 75分员工打开敏感文件 → 70分（-5分），需确认
- 65分员工插入U盘 → 65分（不变）

---

#### 3️⃣ 低信任员工（<60分）

**设计理念**: 严格管控，防止数据泄露

| 行为类型 | 操作 | 扣分 | 前端响应 |
|---------|------|------|---------|
| 敏感文件访问 | `confirm_required` | -15分 | 强制两按钮弹窗 |
| USB插入 | `confirm_required` | -5分 | 二次确认弹窗 |
| 正常办公 | `pass` | 0分 | 无响应 |

**示例**:
- 50分员工打开敏感文件 → 35分（-15分），强制确认
- 45分员工插入U盘 → 40分（-5分），需确认

---

### 特殊规则：远程控制软件

无论信任分多少，都直接阻断（`block`），但扣分幅度不同：

| 信任分区间 | 扣分 | 操作 | 说明 |
|-----------|------|------|------|
| <60分 | -20分 | block | 严惩，直接杀进程 |
| 60-80分 | -10分 | block | 中等惩罚，直接杀进程 |
| >80分 | -5分 | block | 轻度惩罚，直接杀进程 |

**示例**:
- 88分员工运行向日葵 → 83分（-5分），立即杀进程
- 50分员工运行TeamViewer → 30分（-20分），立即杀进程

---

## 💡 设计原则

### 三级响应策略

1. **高信任员工（>80分）**
   - ✅ 轻度惩罚（-2分）
   - ✅ 仅告警不强制确认
   - ✅ 仍执行阻断保护
   - **目的**: 避免过度干扰正常工作

2. **中信任员工（60-80分）**
   - ⚠️ 中等惩罚（-5分）
   - ⚠️ 需要二次确认
   - ⚠️ 管理员可审批
   - **目的**: 平衡安全与效率

3. **低信任员工（<60分）**
   - 🛑 重度惩罚（-15分）
   - 🛑 强制弹窗确认
   - 🛑 严格监管
   - **目的**: 防止数据泄露

## 🔧 代码实现

### 后端决策矩阵

**文件**: `backend/api/behavior.py`（第544-552行）

```python
elif trust_score > 80 and behavior_risk == 'high':
    action_taken = "warn"
    alert_level = "medium"
    new_trust_score = max(new_trust_score - 2, 0)
    try:
        print(f"📝 [矩阵决策] 高信任({trust_score:.1f}) + 高危行为 → 仅告警，信任分-2")
    except OSError:
        print(f"[Matrix Decision] High trust({trust_score:.1f}) + High risk behavior → Warning only, trust score -2")
```

### 前端阻断执行

**文件**: `frontend_agent/collector.py`（第1390-1408行）

```python
# ② 文件操作类：锁文件 + 关tab + 全屏遮罩
if target_file and os.path.exists(target_file):
    # 先锁文件权限
    r = terminal_blocker.block_file_access(
        target_file,
        reason=file_op.get('operation_detail', '敏感文件操作')
    )
    results.append(r)
    print(f"   已锁定文件: {target_file}")

    # 立刻关闭当前编辑器标签页
    terminal_blocker.close_current_editor_tab()

    # 显示全屏阻断遮罩
    terminal_blocker.show_blocking_overlay(
        message="敏感文件操作已被阻断",
        sub_message=f"文件: {os.path.basename(target_file)}\n该文件已被锁定，无法再次打开"
    )
```

## 📈 审计日志

每次信任分变更都会记录到审计日志：

```python
audit_logger.log_trust_score_update(
    user_id=user.id,
    old_score=88.09,
    new_score=86.09,
    reason='敏感词违规'
)
```

**查询方式**:
```sql
SELECT * FROM audit_logs 
WHERE log_type = 'permission' 
  AND action = 'trust_score_update'
ORDER BY created_at DESC;
```

## 🎓 总结

### 核心要点

1. **分级响应**: 根据信任分区间采用不同的惩罚力度
2. **高信任宽容**: >80分员工仅扣2分，避免过度干扰
3. **仍然阻断**: 即使只是warn，前端仍执行文件锁定
4. **审计追溯**: 所有变更都有日志记录

### 优势

- ✅ **安全性**: 敏感文件始终被阻断
- ✅ **用户体验**: 高分员工不被过度惩罚
- ✅ **可追溯**: 完整的审计日志
- ✅ **灵活性**: 可根据实际情况调整阈值

### 适用场景

- 企业零信任安全系统
- 数据防泄露（DLP）
- 员工行为监控
- 合规审计
