# 功能验证指南

本文档说明如何验证已实现的两个核心功能：
1. **孤立森林模型自动训练**
2. **真实终端阻断功能**

---

## ✅ 功能1：孤立森林模型自动训练

### 实现内容
- ✅ 应用启动时自动训练孤立森林模型
- ✅ 每24小时定时重新训练模型
- ✅ 修复了 behavior.py 中变量顺序问题

### 验证步骤

#### 步骤1：启动后端服务
```bash
cd backend
python app.py
```

#### 步骤2：观察启动日志
启动后应该看到类似以下输出：
```
🚀 系统启动中...

============================================================
🤖 开始训练孤立森林模型...
============================================================
[Isolation Forest] 训练完成，样本数: XXX
✅ 孤立森林模型训练成功！
============================================================

✅ 孤立森林模型训练任务已启动（启动时训练 + 每24小时重训）
```

#### 步骤3：检查后台线程是否运行
在Python控制台或通过任务管理器确认有两个后台线程在运行：
- `train_isolation_forest_background` - 启动时训练
- `scheduled_retrain_iforest` - 定时重训

#### 步骤4：模拟数据不足场景
如果数据库中行为日志少于50条，会看到：
```
⚠️ 孤立森林模型训练失败（数据不足），将在首次检测时重试
```

这是正常的，当有足够数据后会自动训练。

#### 步骤5：触发异常检测
通过前端模拟器或Agent上报行为数据，观察日志中是否有：
```
[Isolation Forest] 预测异常: ...
```

如果有，说明模型正在工作。

## 🔍 常见问题排查

### 问题1：孤立森林模型训练失败
**症状**：看到 "数据不足，跳过训练"

**原因**：数据库中行为日志少于50条

**解决**：
1. 通过前端模拟器多发送一些行为数据
2. 或者导入测试数据：
```bash
cd database
mysql -u root -p zero_trust_office < mock_data.sql
```

### 问题2：阻断功能执行失败
**症状**：看到 "可能需要管理员权限"

**原因**：没有以管理员/root身份运行

**解决**：
- Windows：右键PowerShell → "以管理员身份运行"
- macOS：使用 `sudo python collector.py`

### 问题3：网络阻断后无法恢复
**症状**：执行 restore_network() 后网络仍未恢复

**原因**：可能需要手动启用网络适配器

**解决**：
- Windows：控制面板 → 网络和共享中心 → 更改适配器设置 → 右键启用
- macOS：系统偏好设置 → 网络 → 点击齿轮 → 使服务激活
