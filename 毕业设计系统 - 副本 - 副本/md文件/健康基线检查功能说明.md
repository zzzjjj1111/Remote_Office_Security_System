# 终端健康基线检查功能 - 修改说明与验证指南

## 📋 修改总结

### 问题描述
豆包指出的缺陷：**能采集补丁/杀毒状态，但无合规判断、无强制检查**

### 修改前后对比

| 模块 | 修改前 | 修改后 |
|------|--------|--------|
| **Agent采集** | 只采集应用、窗口、CPU等基础信息 | ✅ 新增补丁状态和杀毒软件状态采集 |
| **Device模型** | 只有mac_address和user_id | ✅ 新增patch_status、antivirus_status、compliance_status、last_health_check |
| **合规判定** | ❌ 完全没有 | ✅ 自动判断合规/警告/不合规三级状态 |
| **告警联动** | ❌ 不合规无告警 | ✅ 自动生成健康检查告警，降低信任分 |
| **前端展示** | 只显示设备基本信息 | ✅ 展示补丁、杀毒、合规状态和趋势图 |
| **强制检查** | ❌ 无定期检查 | ✅ 每次行为上报自动触发健康检查 |

## 🔧 详细修改清单

### 1. Agent端 (collector.py)

**新增函数：**
- `get_patch_status()`: 检测Windows/Mac补丁状态
- `get_antivirus_status()`: 检测杀毒软件状态

**修改函数：**
- `collect_behavior()`: 返回数据新增`patch_status`和`antivirus_status`字段

**判断逻辑：**
```
Windows补丁状态：
- 90天内≥3个更新 → "已更新"
- 90天内1-2个更新 → "缺失关键补丁"
- 90天内0个更新 → "严重落后"

macOS补丁状态：
- macOS 14.x+ → "已更新"
- macOS 13.5+ → "缺失关键补丁"
- 更低版本 → "严重落后"

杀毒软件状态：
- 检测到运行中的杀毒软件 → "XXX 正常"
- 检测到但未启用 → "XXX 过期"
- 未检测到 → "未安装/未运行"
```

### 2. 后端数据库模型 (db.py)

**Device表新增字段：**
```python
os_info = db.Column(db.String(100))  # 操作系统信息
patch_status = db.Column(db.String(255))  # 补丁状态
antivirus_status = db.Column(db.String(255))  # 杀毒软件状态
last_login_time = db.Column(db.DateTime)  # 最后登录时间
created_at = db.Column(db.DateTime)  # 设备创建时间
last_health_check = db.Column(db.DateTime)  # 最后健康检查时间
compliance_status = db.Column(db.Enum('compliant', 'warning', 'non_compliant'))  # 合规状态
```

**⚠️ 重要：需要执行数据库迁移**
```sql
USE zero_trust_office;

-- 为现有devices表添加缺失字段
ALTER TABLE devices 
ADD COLUMN os_info VARCHAR(100) COMMENT '操作系统信息',
ADD COLUMN last_login_time DATETIME COMMENT '最后登录时间',
ADD COLUMN created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '设备创建时间',
ADD COLUMN last_health_check DATETIME COMMENT '最后健康检查时间',
ADD COLUMN compliance_status ENUM('compliant', 'warning', 'non_compliant') DEFAULT 'compliant' COMMENT '合规状态';
```

### 3. 后端服务层 (health_check_service.py) ⭐ 核心新增

**合规判定规则：**

| 补丁状态 | 杀毒状态 | 合规状态 | 风险分数 | 触发动作 |
|---------|---------|---------|---------|---------|
| 已更新 | 正常 | compliant | 0 | pass |
| 缺失关键补丁 | 正常 | warning | -10 | warn |
| 已更新 | 过期 | warning | -15 | warn |
| 缺失关键补丁 | 过期 | warning | -25 | warn |
| 严重落后 | 任意 | non_compliant | -30 | block |
| 任意 | 未安装/未运行 | non_compliant | -25 | block |

**信任分扣除规则：**
- non_compliant: 信任分 -5
- warning: 信任分 -2

---

### 4. 后端API (health.py) ⭐ 核心新增

**新增API端点：**

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/health/check` | POST | 设备健康检查 |
| `/api/health/stats` | GET | 合规统计 |
| `/api/health/devices/<id>` | GET | 单个设备健康详情 |
| `/api/health/devices` | GET | 设备列表（分页+过滤） |
| `/api/health/compliance-history` | GET | 合规趋势历史 |

**集成到行为上报：**
- 修改`behavior.py`的`/upload`接口
- Agent上报时自动触发健康检查
- 不合规设备自动生成告警

## 🐛 常见问题排查

### 问题1：Agent采集不到补丁状态
**原因：** WMI权限不足或wmi模块未安装
**解决：**
```bash
pip install wmi pywin32
# 以管理员身份运行collector.py
```

### 问题2：数据库字段缺失错误
**原因：** 未执行ALTER TABLE语句
**解决：** 执行上方"验证2"中的SQL

### 问题3：前端页面报错"Module not found"
**原因：** 未安装echarts
**解决：**
```bash
cd frontend_web
npm install echarts
```

### 问题4：健康检查API返回500错误
**原因：** 数据库连接问题或Device表字段不存在
**解决：**
1. 检查backend/core/config.py中的数据库配置
2. 确认已执行数据库迁移SQL
